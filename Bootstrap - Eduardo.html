Alertas (Alerts)

Botões (Buttons)

Use os estilos de botão personalizados do Bootstrap para ações em formulários, diálogos e muito mais, com suporte para vários tamanhos, estados e muito mais. O Bootstrap inclui vários estilos de botões pré-definidos, cada um com sua própria finalidade semântica, com alguns extras adicionados para maior controle.

<button type="button" class="btn btn-primary">Primário</button>
<button type="button" class="btn btn-secondary">Secundário</button>
<button type="button" class="btn btn-success">Sucesso</button>
<button type="button" class="btn btn-danger">Perigo</button>
<button type="button" class="btn btn-warning">Cuidado</button>
<button type="button" class="btn btn-info">Informação</button>
<button type="button" class="btn btn-light">Claro</button>
<button type="button" class="btn btn-dark">Escuro</button>
<button type="button" class="btn btn-link">Link</button>

Botão com contorno

Quer um botão, mas sem as cores de fundo? Substitua as classes modificadoras padrão por .btn-outline- * para remover todas as imagens e cores de fundo de qualquer botão.

<button type="button" class="btn btn-outline-primary">Primário</button>
<button type="button" class="btn btn-outline-secondary">Secundário</button>
<button type="button" class="btn btn-outline-sucess">Sucesso</button>
<button type="button" class="btn btn-outline-danger">Perigo</button>
<button type="button" class="btn btn-outline-warning">Aviso</button>
<button type="button" class="btn btn-outline-info">Informação</button>
<button type="button" class="btn btn-outline-light">Claro</button>
<button type="button" class="btn btn-outline-dark">Escuro</button>

Botões com diferentes tamanhos

Existem diferentes tamanhos de botões que você pode utilizar, temos o botão grande que para ser utilizado você deve declarar o valor ".btn-lg" "lg" vem do inglês "large" que significa grande. Temos o botão com o tamanho padrão que não é necessario declarar nada. Temos o botão pequeno que para ser utilizado deve ser declarado o valor ".btn-sm" "sm" vem do inglês "small" que significa pequeno, e temos o botão largo que para ser utilizado deve-se declarar o valor ".btn-block". O botão com largura total ocupa uma linha inteira da tela.

Botão Grande

<button type="button" class="btn btn-primary btn-lg">Botão Grande</button>

Botão Padrão
<button type="button" class="btn btn-primary">Botão Padrão</button>

Botão Pequeno
<button type="button" class="btn btn-primary btn-sm">Botão Pequeno</button>

Botão Largo
<button type="button" class="btn btn-primary btn-block">Botão Largo</button>

Botão Ativo

Os botões aparecerão pressionados (com um fundo mais escuro, borda mais escura e sombra inserida) quando ativos. Não é necessario adicionar nada ao codigo pois eles já vem com a função ativa por padrão.

<button type="button" class="btn btn-primary">Botão Ativo</button>
Botão Desativado

Faz o botão parecer inativo adicionando o atributo booleano "disable" a qualquer elemento "button".

<button type="button" class="btn btn-primary" disabled>Botão Desativado</button>

Botão com Spinner

Para que o botão possua um spinner, é necessário declarar uma classe própria para isso, cujo está representada logo abaixo. Note que trata-se de um spinner pequeno, por isso temos aquela mesma declaração "sm", vista anteriormente.

<button type="button" class="btn btn-primary">
  <span class="spinner-border spinner-border-sm"</span>
  Carregando...
</button>

Carrossel (Carousel)

O carrossel é uma apresentação de slides para percorrer uma série de conteúdo, construída com transformações CSS 3D e um pouco de JavaScript. Ele funciona com uma série de imagens, texto ou marcação personalizada. Também inclui suporte para controles e indicadores anteriores/futuros.

Com controle e indicador

Aqui está um carrossel com controle e indicador de imagem. Observe a presença do .d-block e do fluido -img nas imagens do carrossel para impedir o alinhamento da imagem padrão do navegador.

A classe .active precisa ser usada em um dos slides, caso contrário, o carrossel não ficará visível. Também, se assegure de definir um único ID no .carousel para os controles opcionais, especialmente, se você está usando múltiplos carousels em uma página. Elementos de controle e indicador devem ter um atributo data-target (ou links href) que combinam com o ID do elemento .carousel.

Grade (Grid)

O sistema de grade do Bootstrap usa uma série de contêiners, linhas e colunas para layout e alinhamento do conteúdo. É construído com flexbox e é totalmente responsivo.

Grade com trêvalue colunas iguais

Este grid é um modelo simples e em seu código temos: row (elementos para envolver colunas), col-sm-4 ("col" se refere a "coluna", "sm"(small) ao seu tamanho e "4" a sua largura) e background-color (cor de fundo das colunas). Vale ressaltar que o "4" está nas trêvalue colunas para que elas tenham o mesmo tamanho, mas isso não é obrigatório.

</div>

Input

O Input é usado para criar controles interativos para formulários com intuito de receber dados do usuário. A seguir, veremos dois exemplos desse componente do Bootstrap.

Login

Esse exemplo de input é de uma tela de Login onde o usuário digita seu endereço de email e senha. E, logo em seguida temos um botão que possui a característica "submit" que serve para executar uma ação. Porém, ele não está ligado a nenhuma outra página ou recurso para que o mesmo funcione, logo, ao apertar em "Entrar" dará erro por esse motivo, servindo, assim, apenas para ser visualizado.

<form action="/action_page.php">
  <div class="form-group">
     <label for="usr">Usuário:</label>
     <input type="text" class="form-control" id="usr" name="username">
  </div>
  <div class="form-group">
     <label for="pwd">Senha:</label>
     <input type="password" class="form-control" id="pwd" name="password">
  </div>
  <button type="submit" class="btn btn-primary">Entrar</button>
</form>

Caixa de Texto

Esse exemplo de input trata-se de uma caixa de comentário. Com o botão de "Feito" ocorre o mesmo que com o botão do exemplo acima, portanto, servindo apenas para visualização.

<form action="/action_page.php">
  <div class="form-group">
     <label for="comment">Comente:</label>
     <textarea class="form-control" rows="5" id="comment" name="text"></textarea>
  </div>
  <button type="submit" class="btn btn-primary">Comentar</button>
</form>

Navbar

Os links de navegação da Navbar baseiam-se em nossas opções .nav com sua própria classe de modificadores e exigem o uso de classes toggler para um estilo responsivo adequado. A navegação nas navbars também aumentará para ocupar o máximo de espaço horizontal possível para manter o conteúdo da barra de navegação alinhado de forma segura.

Estados ativos - com .active - para indicar a página atual podem ser aplicados diretamente a .nav-links ou a seus pais imediatos .nav-item.

Navbar

  • Desativado

Tabelas

A tabela serve para imprimir informações property tela, onde se é usada a classe própria para tabela (class="table"). A tag "th" serve para definir o que será o cabeçalho da tabela. Uma tabela é dividida em linhas com a tag "tr" e cada linha é dividida em células de dados com a tag "td" (significa "table data") que é o conteúdo de uma célula de dados. A tag "tbody" define o corpo de dados de uma tabela. O uso de TBODY é obrigatório sempre que utilizamos THEAD ou TFOOT. Sua omissão valueó é permitida quando não forem definidos os cabeçalhos e os rodapévalue, e apenas existir um corpo de dados.
